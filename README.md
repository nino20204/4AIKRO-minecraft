📖 نظرة عامة (Overview)
نظام متجر إلكتروني متكامل ومتقدم مبني بـ HTML5, CSS3, JavaScript، مصمم خصيصاً لسيرفرات ماينكرافت وروبلوكس. يتميز النظام ببنية بيانات قوية، نظام مخزون متطور، ودعم كامل لتعدد اللغات (العربية، الإنجليزية، اليابانية) مع حماية عالية لبيانات المستخدمين.

✨ المميزات الرئيسية (Key Features)
1. نظام إدارة المستخدمين (User Management)
تسجيل الدخول وإنشاء الحساب: نظام تسجيل دخول آمن باستخدام Username و Secret Code.
استرجاع الحساب: إمكانية استرجاع الحساب عبر البريد الإلكتروني.
جلسات المستخدم: الحفاظ على تسجيل الدخول باستخدام Local Storage.
2. نظام المخزون المتطور (Inventory System)
نظام التكديس (Stacking): دمج تلقائي للعناصر المكررة في خانة واحدة (مثلاً: مفتاح x2 بدلاً من خانتين).
الحماية من فقدان البيانات: آلية Sanitization تعمل تلقائياً عند فتح المخزون لإصلاح البيانات التالفة أو القديمة.
الترقية التلقائية (Auto-Migration): تحويل البيانات من الأنظمة القديمة (مثل مفاتيح mk أو inv) إلى النظام الجديد inventory بشكل سلس.
عرض التفاصيل: نظام ذكي يعتمد على ID لعرض التفاصيل، مما يمنع حدوث أخطاء عند إعادة ترتيب العناصر.
3. نظام الهدايا والإحالة (Referrals & Gifts)
هدية يومية: نظام هدايا يومية مع مؤقت عد تنازلي (Cooldown Timer).
نظام الإحالة: رابط دعوة فريد لكل مستخدم، مع مكافآت للإحالات الجديدة (يتم اضافتها تلقائياً للمخزون).
حد يومي: تحديد عدد مرات الإحالة المسموحة يومياً (5 مرات افتراضياً).
4. واجهة المستخدم (UI/UX)
تصميم عصري: تصميم "Glassmorphism" مع تأثيرات خلفية متحركة.
تأثيرات التحميل: تأثير Shimmer للصور أثناء التحميل لتحسين تجربة المستخدم.
تتوافقية: تصميم متجاوب يعمل بشكل ممتاز على الهواتف والحواسيب.
الوضع الليلي: تصميم داكن مريح للعين.
5. تعدد اللغات (Multi-Language Support)
دعم كامل للغات: العربية (AR)، الإنجليزية (EN)، اليابانية (JP).
تبديل اللغة فوري دون إعادة تحميل الصفحة.
دعم اتجاه النص (RTL للعربية، LTR للغات الأخرى).
🛡️ الأمان وحماية البيانات (Security & Data Integrity)
مشاكل تم حلها في هذا الإصدار:
إصلاح مشكلة NaN: تحويل القيم غير الصحيحة للأرقام إلى 0 أو 1 تلقائياً.
منع التكرار: دمج العناصر المتشابهة لمنع تضخم قاعدة البيانات.
معالجة الترتيب: فصل منطق العرض عن ترتيب المصفوفة لضمان ظهور التفاصيل الصحيحة دائماً.
التراجع عن الأخطاء: إذا فشل تحميل بيانات المستخدم، يقوم النظام بإنشاء بيانات افتراضية آمنة بدلاً من تعطل الموقع.
